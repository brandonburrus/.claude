# hooks/

Hook scripts executed by the Claude Code harness on session events. Each script is wired in via the `hooks` block in `../settings.json`; the script file is re-read per invocation, but the settings wiring only loads at session start, so wiring changes need a fresh session to take effect.

## Conventions

- Language by weight: bash + `jq` for short pattern checks (the PreToolUse Bash guards), Python stdlib + system `python3` when logic outgrows jq (the AGENTS.md injector). Neither uses `uv run`: hooks fire on every matching tool call and must not pay dependency-resolution latency.
- Wire bash hooks via `bash ~/.claude/hooks/<name>.sh` and Python via `python3 ~/.claude/hooks/<name>.py`: the explicit interpreter means the scripts never need an executable bit (`chmod` is denied by this repo's permissions anyway).
- Convenience hooks fail open (exit 0 on any internal error); security hooks fail closed. State the choice in the script docstring.
- Hooks parse stdin as JSON, never regex the raw stream. Event stdin shapes and output schemas are documented in `../skills/create-claude-hook/references/hook-events.md`.
- Per-session state goes under `~/.cache/claude-agents-md/` style cache dirs, never inside this repo.

## Scripts

- `inject-agents-md.py` - Points Claude at AGENTS.md files (Claude Code natively loads only CLAUDE.md). Root AGENTS.md at SessionStart via stdout; nested ones lazily on PostToolUse (Read|Edit|Write|NotebookEdit) via additionalContext, walking the touched file's ancestry up to cwd. It emits a read-this pointer, not the file's content: Claude reliably opens an AGENTS.md once told one is nearby, so inlining only duplicated context and risked serving a stale copy. Dedup is per session per file via path+mtime state keyed by session_id plus transcript basename (isolates subagents from the main loop); compaction clears the state so files re-surface. Test battery pattern: pipe hand-built event JSON through the script and assert stdout/exit codes.
- `enforce-verification.py` - PostToolUse(Edit|Write|NotebookEdit|Bash) + Stop. Pushes the main loop to verify code it changes. An edit to a source file (app-code extensions only; markdown/config/shell/SQL excluded so the gate is not trained around) records a pending unverified change, and runs a fast linter the project opted into (ruff with a ruff/pyproject config, biome with a biome config) on just that file as advisory additionalContext. A Bash command matching a test/build/typecheck/lint runner clears the pending set. On Stop, a non-empty pending set blocks the stop once (honoring `stop_hook_active` so it never wedges) with a reason naming the unverified files. The gate enforces that verification was ATTEMPTED, not that it passed (green-but-wrong is the completion-verifier agent's and the CLAUDE.md done-gate's job); delegated work is gated by the subagents' own contracts because subagent edits land under the subagent transcript's own state. Convenience-grade, fails open. State under `~/.cache/claude-verify/`. Test battery: pipe hand-built event JSON through it and assert pending-state transitions, the Stop block JSON, and exit codes.
- `block-interactive-cmds.sh` - PreToolUse(Bash) guard. Denies `cp`/`mv`/`rm` at a statement boundary without an `-f` flag, because macOS aliases them to the `-i` interactive variants and the agent then hangs on an unanswerable prompt. Bypasses (`command rm`, absolute path, backslash) are allowed through. Fails open: non-matching or unparseable input exits 0.
- `block-gh-watch.sh` - PreToolUse(Bash) guard. Denies `gh run watch` / `gh run list --watch`, which poll every ~3s and can exhaust the 5000/hr GitHub API quota; the denial reason points the agent at a single `gh run view`. Fails open.
- `block-no-verify.sh` - PreToolUse(Bash) guard. Denies `git --no-verify` (and the `-n` short form immediately after `commit`), which skip the pre-commit and pre-push hooks. `git push -n` (dry-run, not no-verify) is allowed, and a `-n` appearing inside a commit message is deliberately not matched, because a guard that blocks real commit messages trains the operator to disable it. Fails open.
- `scan-read-for-injection.py` - PostToolUse(Read|WebFetch) scanner. Inspects ingested file and web content for prompt-injection markers (override phrases, summarization-survival instructions, credential-bearing or javascript/data markdown links, invisible unicode) and emits an advisory additionalContext warning at LOW (1-2 hits) or HIGH (3+). Never blocks: a match is a signal, not a verdict, and docs and security skills legitimately carry these strings. Excludes this repo's own `_refs`, security skills, and hooks, which are saturated with example payloads. Fails open. Test battery: pipe hand-built Read/WebFetch event JSON through it and assert the additionalContext and exit code.
- `block-secret-writes.py` - PreToolUse(Write|Edit|NotebookEdit|Bash) secret guard. Denies writing a recognizable credential (provider key prefixes such as `sk-`/`ghp_`, AWS `AKIA` key ids, private-key headers) into a file or a shell command; the deny reason routes to an env var or secrets manager and says rotate-do-not-relocate for a live-looking key. Only high-precision prefixed shapes are matched (generic entropy is omitted so the guard stays trusted when it fires), and placeholder tells (`EXAMPLE`, `your-api-key`, `<...>`) are skipped so `.env.example` templates pass. It deliberately fails OPEN, the one security hook here that does: the threat model is accidental leakage by the sole operator, not an adversary crafting malformed envelopes, so a parser bug must never wedge every write. The reason is in its docstring; do not "fix" it to fail closed.
