When working with git, ALWAYS follow these rules:

- When you don't know the codebase git commit conventions, review the most recent 5 commits and follow the established patterns. If there is no established pattern or the existing pattern is eratic, follow conventional commits.
- Prefer short and direct commit messages that convey the "what" of what is being commited based on the code itself. Avoid commits like "addressing PR comments" or generic "refactored code" commit messages.
