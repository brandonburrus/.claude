## D3 reference

When to reach for D3 (v7), the standard chart skeleton, scales, interactivity, and responsiveness. D3 binds data to the DOM and gives fine-grained control over every SVG element, for custom visualizations beyond what charting libraries offer. Condensed from the d3-viz pattern reference.

### When D3 is the right tool

Use D3 for custom encodings or layouts, network and graph diagrams (force, tree, hierarchy, chord), geographic maps with custom projections, choreographed transitions, novel chart types, and publication-quality control over every element. For a standard bar, line, or scatter that a library or `st.*_chart` already covers, use that; D3 earns its complexity only when no library does what you need. For 3D, use Three.js.

### Setup

```javascript
import * as d3 from 'd3';            // bundled
// or CDN: <script src="https://d3js.org/d3.v7.min.js"></script>
```

Two integration patterns: (A) D3 owns the DOM, selecting and appending elements imperatively, best for transitions and interactions; (B) D3 computes scales and layouts and your framework (React, Vue, Svelte) renders the elements declaratively. Use A for rich interaction, B for a simple chart inside a component tree.

### The standard chart skeleton

Every chart follows the same shape: dimensions and margins, a translated group, scales, axes, then a data join.

```javascript
function drawChart(data, svgEl) {
  if (!data?.length) return;
  const svg = d3.select(svgEl);
  svg.selectAll("*").remove();                 // clear previous render

  const width = 800, height = 400;
  const margin = { top: 20, right: 30, bottom: 40, left: 50 };
  const iw = width - margin.left - margin.right;
  const ih = height - margin.top - margin.bottom;

  const g = svg.append("g")
    .attr("transform", `translate(${margin.left},${margin.top})`);

  const x = d3.scaleBand().domain(data.map(d => d.category)).range([0, iw]).padding(0.1);
  const y = d3.scaleLinear().domain([0, d3.max(data, d => d.value)]).range([ih, 0]); // inverted for SVG

  g.append("g").attr("transform", `translate(0,${ih})`).call(d3.axisBottom(x));
  g.append("g").call(d3.axisLeft(y));

  g.selectAll("rect").data(data).join("rect")   // .join handles enter/update/exit
    .attr("x", d => x(d.category))
    .attr("y", d => y(d.value))
    .attr("width", x.bandwidth())
    .attr("height", d => ih - y(d.value))
    .attr("fill", "steelblue");
}
```

The y range is inverted (`[ih, 0]`) because SVG y grows downward. Use `.join()`, never hand-rolled enter/update/exit.

### Scales

| Need | Scale |
|---|---|
| Continuous numbers | `d3.scaleLinear` (`scaleLog` / `scalePow` for skew) |
| Time | `d3.scaleTime` |
| Categories to bands (bars) | `d3.scaleBand` (`scalePoint` for line/scatter categories) |
| Category colours | `d3.scaleOrdinal(d3.schemeCategory10)` |
| Value to colour (heatmap) | `d3.scaleSequential(d3.interpolateBlues)` |
| Diverging value to colour | `d3.scaleDiverging(d3.interpolateRdBu)` |

Each scale maps a `.domain()` (data space) to a `.range()` (pixel or colour space).

### Patterns

- **Line:** `const line = d3.line().x(d => x(d.date)).y(d => y(d.value)).curve(d3.curveMonotoneX);` then `g.append("path").datum(data).attr("fill","none").attr("stroke","steelblue").attr("d", line);`
- **Scatter:** join `circle`, set `cx`/`cy` from scales, optional `r` from a size scale and `fill` from a colour scale.
- **Heatmap:** two `scaleBand` axes, a `scaleSequential` colour scale, join `rect`.
- **Network:** `d3.forceSimulation(nodes).force("link", d3.forceLink(links).id(d => d.id)).force("charge", d3.forceManyBody()).force("center", d3.forceCenter(w/2, h/2))`, update node and link positions in `simulation.on("tick", ...)`.
- **Chord, pie, tree, geographic:** the same skeleton with the matching layout generator (`d3.chord`, `d3.pie` + `d3.arc`, `d3.tree` / `d3.hierarchy`, `d3.geoPath` + a projection).

### Interactivity

- **Tooltip:** a positioned `div` outside the SVG, toggled on `mouseover` / `mouseout` and moved on `mousemove`.
- **Zoom and pan:** `svg.call(d3.zoom().scaleExtent([0.5, 10]).on("zoom", e => g.attr("transform", e.transform)))`.
- **Click and hover:** `.on("click", (event, d) => ...)`; D3 passes `(event, datum)`.
- **Transitions:** `sel.transition().duration(750).attr("r", 10)`; stagger with `.delay((d, i) => i * 50)`.

### Responsiveness

Watch the container and redraw on resize:

```javascript
const ro = new ResizeObserver(() => {
  const { width } = svgEl.parentElement.getBoundingClientRect();
  drawChart(data, svgEl, width);   // recompute scales from the new width
});
ro.observe(svgEl.parentElement);
```

Or set a `viewBox` and let the SVG scale (simplest, but text scales with it).

### Accessibility and performance

- `svg.attr("role", "img").attr("aria-label", "...")`, add `<title>` and `<desc>`, ensure colour contrast, and offer a data-table alternative.
- Beyond about a thousand elements, switch from SVG to canvas, debounce resize handlers, and rely on `.join()` to avoid redundant DOM churn.
